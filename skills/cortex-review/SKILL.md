# Cortex Review — Skill

## When to Use
When reviewing code, designs, or proposals.

## Workflow

1. **Query cortex** for the topic being reviewed
2. **Read the cortex results** — they contain past decisions, patterns, and lessons
3. **Compare** the work under review against cortex context
4. **Flag divergences** — where does the new work differ from established patterns?
5. **Validate alignment** — where does it correctly extend existing patterns?

## Review Output Format

```
## Cortex-Grounded Review

### Aligned with existing patterns:
- [list items that match cortex context]

### Divergences to discuss:
- [list items that differ from cortex context, with rationale]

### New patterns to observe:
- [list novel approaches worth capturing via observe()]
```
